## 📌 DB Join이란?
- 여러 테이블을 공통 컬럼(주로 외래키)을 기준으로 결합하는 SQL 명령어  
- 정규화된 데이터에서 분산된 정보를 하나의 결과로 조회  
- **장점**: 중복 제거, 무결성 유지, 유연한 조회  
- **단점**: 쿼리 복잡, 성능 저하 가능  

### Join 종류
- **INNER JOIN**: 교집합, 양쪽에 모두 있는 데이터만 조회  
- **LEFT JOIN**: 왼쪽 테이블 기준, 없는 데이터는 NULL  
- **RIGHT JOIN**: 오른쪽 테이블 기준, 없는 데이터는 NULL  
- **FULL OUTER JOIN**: 합집합, 매칭 안 되면 NULL (MySQL 미지원)  
- **CROSS JOIN**: 모든 조합 생성  
- **SELF JOIN**: 자기 자신과 조인, 계층 구조 조회

## 📌 트랜잭션(Transaction)
- 데이터베이스의 하나의 논리적 작업 단위  
- **ACID 속성**
  - **원자성**: 전부 성공 or 전부 실패  
  - **일관성**: 제약조건 유지  
  - **격리성**: 동시에 실행돼도 서로 간섭 없음  
  - **지속성**: 커밋된 결과는 영구 저장  
- **장점**: 무결성 보장, 동시성 제어, 오류 복구 가능  
- **단점**: 성능 부담, 데드락 위험, 리소스 점유  

---

## 📌 JOIN ON vs WHERE
- **ON**: 조인 조건, 조인 시점에 적용 → 조인 결과 자체에 영향  
- **WHERE**: 최종 결과 필터링, 조인 이후에 적용  
- **차이점**: OUTER JOIN에서는 ON은 NULL 유지, WHERE는 행 제거
