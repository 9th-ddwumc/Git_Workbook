### 📚 1주차 학습 내용 정리

### **Git이란?**
**Git**은 프로젝트의 **버전 관리 시스템(VCS, Version Control System)** 
 파일의 변경 이력을 기록하고 관리하여, 특정 시점으로 되돌리거나 여러 사람이 동시에 작업하는 것을 돕는 도구다.


### **Working Directory와 Local Repository**
Git에서 작업하는 공간은 크게 두 가지!
* **Working Directory (작업 디렉토리)**: 실제 파일 수정 작업을 하는 폴더
* **Local Repository (로컬 저장소)**: `.git` 폴더가 있는 곳으로, Working Directory의 변경 이력이 저장되는 공간

이 둘 사이에는 **Staging Area(스테이징 영역)**라는 중간 단계가 존재. 수정된 파일을 `git add` 명령어를 통해 이 영역에 올리면 커밋할 준비가 완료!

---

### **원격 레포지토리와 로컬 레포지토리 연결법**
원격 저장소(GitHub)와 로컬 저장소(내 컴퓨터)를 연결하는 방법!

* **`git clone`**: 이미 존재하는 원격 저장소를 내 컴퓨터로 통째로 복사해 올 때 사용.
    ```bash
    git clone [원격 저장소 URL]
    ```

* **`git remote`**: 로컬 저장소에 원격 저장소 주소를 추가하여 연결. 주로 빈 폴더에서 Git 프로젝트를 시작할 때 사용.
    ```bash
    git remote add origin [원격 저장소 URL]
    ```

---

### **`.gitignore`로 파일 관리하기**
`.gitignore` 파일은 Git이 특정 파일이나 폴더를 무시하도록 설정하는 데 사용. 빌드 결과물, 설정 파일, 민감한 정보 등 버전 관리가 필요 없는 것들을 Git 추적 대상에서 제외 가능!

**사용법**:
프로젝트 최상위 폴더에 `.gitignore` 파일을 만들고, 무시할 파일이나 폴더의 이름을 작성합니다.
